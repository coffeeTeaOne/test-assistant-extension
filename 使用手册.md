## 1. 插件简介

**七星瓢虫**是一款 Chrome 扩展（Manifest V3），帮助测试工程师：

| 能力 | 说明 |
|------|------|
| 🎬 **操作录制** | 在网页上的点击、输入、选择等操作自动录制为结构化步骤 |
| 🤖 **AI 智能助手** | 基于 LLM 分析页面功能、设计测试用例、提出产品优化建议 |
| 📝 **脚本生成** | 一键生成 Playwright / Selenium Python 自动化脚本 |
| 📂 **用例管理** | 搜索、筛选、编辑、执行、导出完整测试用例 |
| 🔗 **pytest 集成** | 导出 JSON 步骤文件，直接供 pytest + Playwright 回放 |

## 2. 安装与加载

### 2.1 前置条件

- Chrome 浏览器（推荐 120+ 版本）
- 已安装 Node.js（开发/构建时使用）

### 2.2 构建插件

```bash
# 进入项目目录
cd test-assistant-extension

# 安装依赖
npm install

# 构建生产包
npm run build
```

构建完成后，`dist/` 目录即为插件包。

### 2.3 加载到 Chrome

1. 打开 Chrome → `chrome://extensions/`
2. 右上角开启「**开发者模式**」
3. 点击「**加载已解压的扩展程序**」
4. 选择 `test-assistant-extension/dist/` 文件夹
5. 扩展图标 🐞 出现在 Chrome 工具栏，点击即可打开侧边栏

> 💡 **提示**：若修改代码后需要更新，点击扩展卡片上的「🔄 刷新」按钮即可。

## 4. 用例 Tab — 录制与管理

### 4.1 录制控制区

位于 Tab 顶部，控制录制流程：

| 按钮 | 功能 |
|------|------|
| 🔴 **开始录制** | 在当前页面开始捕获用户操作 |
| ⏹ **停止录制** | 结束录制，保存为用例 |
| **清空** | 清空当前实时步骤预览 |

**录制状态指示**：
- 红色脉冲圆点 + 「录制中... 已记录 N 个步骤」表示正在录制
- 若提示「当前页面的录制脚本未加载，请刷新页面后重试」，点击 🔄 刷新当前页面即可

**实时步骤预览**：录制过程中，每一步操作会实时显示在录制区下方，包括动作类型、目标元素、输入值等信息。

### 4.2 测试用例列表

录制完成后，用例自动加入列表，按时间倒序排列。

#### 搜索与筛选

- 🔍 **名称搜索框**：输入关键词实时过滤用例名称
- 📍 **URL 筛选器**：下拉选择特定页面 URL，只显示该页面相关的用例
- 点击 ✕ 可一键清空对应筛选条件

#### 用例卡片操作

每个用例卡片包含：

| 操作 | 说明 |
|------|------|
| **▶ 执行** | 在当前标签页逐步回放该用例的所有步骤，带页面高亮和结果反馈 |
| **生成脚本** | 为该用例生成 Python 测试脚本（Playwright 或 Selenium） |
| **JSON 自动化配置下载** | 导出 JSON 步骤文件（同时保存到浏览器下载目录和本地目录） |
| **🔒 保存为登录配置** | 将该用例保存为登录档案，供其他用例执行时前置调用 |
| **▼ 展开步骤详情** | 查看/编辑用例的每一步详细信息 |
| **🗑** | 删除用例 |

#### 步骤编辑

展开用例后，可点击任意步骤字段进行**行内编辑**：
- 选择器（CSS Selector）
- XPath
- 输入值（Value）
- 目标文本（Text）
- ARIA 标签
- URL

点击字段 → 输入新值 → 按 `Enter` 或失焦即保存。

#### 登录配置绑定

每个用例卡片右上角有登录配置下拉框：
- 选择已保存的登录档案后，执行/生成脚本/导出 JSON 时会**自动前置登录步骤**
- 适合需要先登录再执行业务操作的场景

## 5. AI Tab — 智能助手

AI Tab 提供两种对话模式，通过顶部按钮切换：

### 5.1 LLM 对话模式（💬 LLM 测试顾问）

纯对话模式，解答测试技术问题，并**自动获取当前页面上下文**（URL、标题、50 个交互元素、表单详情）。

#### 快捷操作按钮

当对话为空时，界面底部显示三个一键按钮：

| 按钮 | 功能 |
|------|------|
| 📋 **分析当前页面功能** | AI 自动分析页面模块划分、可测试点、风险点 |
| 📝 **生成功能测试用例** | AI 基于页面信息输出完整用例表格（含正向/反向/边界场景） |
| 💡 **产品优化建议** | 从交互体验、文案、可访问性、测试可测性角度提出优化建议 |

点击后自动发送预设 Prompt，AI 会在几秒内返回结构化结果。

#### 导出表格为 Excel

当 AI 回复中包含 Markdown 表格时，消息气泡底部会出现 **📥 可导出 Excel** 按钮：
- 点击即可下载 `.csv` 文件
- 自动去除 Markdown 粗体标记（`**`）
- `<br>` 标签自动转为单元格内换行
- 直接用 Excel / WPS 打开即可

### 5.2 Agent 对话模式（🛠 Agent 测试助手）

具备**工具调用能力**的 AI 模式，可直接操作页面、执行录制、生成脚本。

**支持的工具**：
- `get_page_info` — 获取当前页面详细信息
- `click_element` / `input_text` / `navigate_to` — 直接操作页面元素
- `start_recording` / `stop_recording` — 控制录制
- `execute_recording` — 执行已保存的用例
- `generate_script` — 生成测试脚本
- `export_json` — 导出 JSON 配置文件
- `highlight_element` — 高亮页面元素辅助定位

**使用示例**：
```
"分析当前页面有哪些测试点"
"点击左侧菜单的系统信息"
"在搜索框输入自动化并搜索"
"执行上次录制的登录用例"
```

### 5.3 会话管理

- **新会话 ▼**：点击可切换/创建/删除历史会话
- **+**：快速创建新会话
- 会话自动保存到本地，关闭浏览器后仍可恢复

## 6. 设置 Tab — 配置中心

### 6.1 AI 配置

配置 LLM API 接口信息：

| 字段 | 说明 |
|------|------|
| **提供商** | 预设：OpenAI / Claude / DeepSeek / 智谱 / Kimi / Codex / 自定义 |
| **Base URL** | API 接口地址（选择预设后自动填充） |
| **API Key** | 你的 API 密钥 |
| **模型** | 如 gpt-4o、deepseek-chat、moonshot-v1-8k 等 |

> 💡 **建议**：DeepSeek 和 Kimi 是国内较稳定的选择，DeepSeek 的 `deepseek-chat` 性价比高。

### 6.2 导出配置

| 选项 | 说明 |
|------|------|
| **导出时自动覆盖同名文件** | 勾选后，导出同名 JSON 时会覆盖旧文件 |
| **本地导出目录（可选）** | 点击「📂 选择本地导出目录」指定 pytest 项目的 `recordings` 文件夹 |

**导出行为**：
- 浏览器下载目录：文件直接保存到 Chrome 下载目录根目录（作为备份）
- 本地导出目录：通过 File System Access API 直接写入你指定的本地目录（主力使用）

> 💡 **推荐 workflow**：在设置中选择 `pytest_automation/recordings` 作为本地导出目录，导出 JSON 时两边同时保存，pytest 可直接读取本地目录中的文件执行自动化测试。

### 6.3 登录配置档案

显示所有已保存的登录配置档案：
- 每个档案显示名称、目标 URL、步骤数
- 可展开查看详细步骤
- 可点击 🗑 删除档案
- 档案在用例 Tab 中可作为前置登录步骤绑定到任意用例

## 7. JSON 导出与 pytest 集成

### 7.1 导出 JSON

1. 在**用例 Tab** 中，找到目标用例
2. 若需要先登录，在卡片右上角选择登录配置档案
3. 点击「**JSON 自动化配置下载**」
4. 文件同时保存到：
   - Chrome 下载目录（备份）
   - 本地导出目录（若已配置）

### 7.2 pytest 项目结构

```
pytest_automation/
├── conftest.py          # Playwright browser fixture
├── step_executor.py     # JSON 步骤执行引擎
├── test_recordings.py   # 自动扫描 recordings/*.json 并参数化执行
└── recordings/          # 存放导出的 JSON 用例文件
    └── 登录流程.json
    └── 创建项目.json
```

### 7.3 运行测试

```bash
cd pytest_automation
pytest -v
```

`test_recordings.py` 会自动扫描 `recordings/` 目录下所有 `.json` 文件，每个文件作为一个测试用例参数化执行。

### 7.4 JSON 文件格式

```json
{
  "id": "abc123",
  "name": "登录流程",
  "createdAt": 1717000000000,
  "updatedAt": 1717000000000,
  "steps": [
    {
      "stepIndex": 0,
      "action": "input",
      "target": { "selector": "#username", "xpath": "//input[@id='username']" },
      "value": "admin",
      "url": "http://localhost:8086/login",
      "timestamp": 1717000001000
    },
    {
      "stepIndex": 1,
      "action": "click",
      "target": { "selector": "#login-btn", "xpath": "//button[@id='login-btn']" },
      "url": "http://localhost:8086/login",
      "timestamp": 1717000002000
    }
  ]
}
```

## 8. 常见问题

### Q1：点击「开始录制」提示「录制脚本未加载」？

**解决**：当前页面尚未注入 Content Script，点击提示框中的 **🔄 刷新当前页面** 按钮，刷新后再点击开始录制。

### Q2：AI 回复很慢或超时？

**解决**：
- 检查 AI 配置中的 API Key 是否正确
- 尝试切换其他提供商（如 DeepSeek、Kimi）
- Agent 模式下若工具调用过多，可简化指令

### Q3：导出的 JSON 文件 pytest 读不到？

**解决**：
- 确认 pytest 项目的 `recordings/` 目录存在
- 在设置中配置「本地导出目录」为 `pytest_automation/recordings`
- 确认文件名以 `.json` 结尾

### Q4：执行用例时某一步失败了？

**解决**：
- 展开用例查看该步骤的选择器和 XPath
- 尝试编辑步骤，使用更稳定的定位方式（如 id、data-testid）
- 在 AI Tab 中使用 Agent 模式的 `highlight_element` 工具验证元素定位

### Q5：CSV 导出后 Excel 显示乱码？

**解决**：导出的 CSV 已包含 UTF-8 BOM，Excel 直接打开即可正确显示中文。若仍乱码，尝试用「数据 → 从文本/CSV 导入」功能选择 UTF-8 编码。

### Q6：本地导出目录权限丢失？

**解决**：Chrome 清除站点数据后，File System Access API 的目录权限会失效。重新进入设置 → 点击「📂 选择本地导出目录」重新授权即可。
